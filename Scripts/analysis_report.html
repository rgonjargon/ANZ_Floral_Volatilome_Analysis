Introduction

  • A minimal reproducible example of a simulation suite that demonstrates how environment, genetics, and experimental design play a role in recovering known population parameters and therefore implications for ecological inference.

Overview

The simulation should create an array (multiple matrices) that contains all the simulated trial information. Each matrix will represent a different variable (i.e., in this example “pid”, “site”, “a”, “b”, “x”, “t1”, “t2”, “t3”) that is created by unique function by different team members (with the intent that this will become a targets node): i.e.,

  • Tom will create the functions to simulate environmental covariates (a and b) and the overall DAG (i.e., how all the matrices are related to one another based on the DGP or a DAG)

  • Kathy will create the function to explore different trial designs + the number of checks to be included

  • Poppy and Linley will create a function estimate the breeding values based on phylogeny

  • Marcus will oversee the pipeline development (at the moment the pipeline isn’t very flexible).

  • At the end it would be good to have a function that iterates over the pipeline (i.e., similar to a power analysis - but would need to specify critorion to test).

  • Should be flexible to different types of DAGS

  • Should be parameterised in a way to showcase why it is important to consider experimental design, covariates, and genetic relationships when estimating breeding values.

# Introduction

-  A minimal reproducible example of a simulation suite that demonstrates how environment, genetics, and experimental design play a role in recovering known population parameters and therefore implications for ecological inference. 


# Overview

The simulation should create an array (multiple matrices) that contains all the simulated trial information. Each matrix will represent a different variable (i.e., in this example "pid", "site", "a", "b", "x", "t1", "t2", "t3") that is created by unique function by different team members (with the intent that this will become a targets node): i.e.,


- Tom will create the functions to simulate environmental covariates (a and b) and the overall DAG (i.e., how all the matrices are related to one another based on the DGP or a DAG)

- Kathy will create the function to explore different trial designs + the number of checks to be included

- Poppy and Linley will create a function estimate the breeding values based on phylogeny

- Marcus will oversee the pipeline development (at the moment the pipeline isn't very flexible).


- At the end it would be good to have a function that iterates over the pipeline (i.e., similar to a power analysis - but would need to specify critorion to test).
- Should be flexible to different types of DAGS
- Should be parameterised in a way to showcase why it is important to consider experimental design, covariates, and genetic relationships when estimating breeding values.
